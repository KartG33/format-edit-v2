# E-dit — Детальное описание приложения

> PWA-приложение для быстрой очистки и форматирования текста.
> Работает офлайн, устанавливается на домашний экран.

---

## Общее

E-dit — мобильное веб-приложение (PWA), разработанное для быстрой обработки текста прямо в браузере. Основная аудитория — музыканты, работающие с Suno AI, а также все кто регулярно очищает, форматирует и преобразует текстовые данные.

Хостинг: https://github.com/KartG33/format-edit-v2 (GitHub Pages)

---

## Структура репозитория

```
format-edit-v2/
├── index.html       ← всё приложение (~4550 строк: HTML + CSS + JS)
├── manifest.json    ← PWA манифест
├── sw.js            ← Service Worker (офлайн)
└── icons/
    ├── icon-16x16.png
    ├── icon-32x32.png
    ├── icon-192x192.png
    └── icon-512x512.png
```

Вся логика приложения — HTML-разметка, стили и JavaScript — сосредоточена в одном файле `index.html`. Это сознательное решение: нет сборщиков, нет зависимостей, нет node_modules. Чтобы обновить приложение — достаточно заменить один файл.

---

## Технический стек

- **Язык:** чистый HTML + CSS + JavaScript — без фреймворков и библиотек
- **PWA:** `manifest.json` + `sw.js`
- **Хранилище:** `localStorage` (всё хранится локально на устройстве, без сервера)
- **Шрифты:**
  - `Geologica` — UI-элементы, кнопки, метки
  - `JetBrains Mono` — текстовый редактор, теги, код
- **Цветовая тема:** тёмная, индиго/синяя палитра, CSS-переменные

## PWA — установка и офлайн

### manifest.json

```json
{
  "name": "E-dit",
  "short_name": "E-dit",
  "display": "standalone",
  "orientation": "portrait",
  "background_color": "#0b0f1a",
  "theme_color": "#0b0f1a",
  "lang": "ru",
  "categories": ["utilities", "productivity"]
}
```

- `display: standalone` — запускается без браузерного chrome, как нативное приложение
- Иконки: 192×192 и 512×512, с поддержкой `maskable` для Android adaptive icons
- Shortcut «Новый текст» — быстрый запуск с чистым полем

### sw.js — Service Worker

Стратегия **network-first**:
1. При каждом запросе сначала идёт в сеть — пользователь всегда получает свежую версию
2. Успешный ответ кладётся в кеш (`e-dit-v3`)
3. Если сеть недоступна — отдаётся из кеша

Кешируются: `index.html`, `manifest.json`, все иконки.
При обновлении Service Worker старые кеши удаляются автоматически по имени версии.

## Интерфейс

Приложение состоит из нескольких зон, расположенных вертикально:

### Хедер

Всегда видим. Содержит:

- **Статистика текста** — четыре пилюли в реальном времени: символы, слова, предложения, абзацы
- **Кнопка «Пресеты»** — открывает шит управления пресетами и регулярками
- **Кнопка «Шаблоны»** — открывает шит шаблонов текста
- **Файловый баннер** — появляется когда открыт файл, показывает имя и кнопку закрытия
- **Кнопка «Установить»** — появляется автоматически когда браузер готов к установке PWA

### Вкладки форматирования

Три вкладки переключают набор кнопок под ними:

- **Suno** — функции для работы с тегами Suno AI (активна по умолчанию)
- **Presets** — пресеты-последовательности (пользовательские)
- **Commands** — стандартные команды форматирования

### Строка символов (Symbol Bar)

Анализирует текст и показывает найденные специальные символы. Два режима:

- **Detect** — каждый символ кнопкой с количеством вхождений, тап — отмечает для удаления
- **Replace** — поле «найти» и «заменить на», применяет по всему тексту

### Текстовый редактор

Большая `<textarea>` с моноширинным шрифтом. Diff-пилюля показывает изменение количества символов после последней операции.

### Нижняя панель (6 кнопок)

Назад / Вперёд (undo/redo), Вставить, Копировать, Стереть, Заметки.

### Сайдбар заметок (десктоп)

На широких экранах — боковая панель справа с вкладками «Сохранённые» и «История». На мобиле открывается через кнопку «Заметки».

## Функции форматирования (Commands)

15 встроенных функций:

### Пробелы и пунктуация

| Функция | Что делает |
|---------|-----------|
| ×2 пробелы | Заменяет два и более пробела на один |
| Неразрывные | Заменяет неразрывные пробелы `\u00A0` на обычные |
| Обрезать края | Убирает пробелы в начале и конце каждой строки |
| Пробел до знака | Удаляет пробелы перед знаками препинания |
| Пробел после знака | Добавляет пробел после знаков препинания если его нет |
| Убрать пунктуацию | Удаляет все знаки препинания |
| Убрать ( ) [ ] | Удаляет содержимое скобок вместе со скобками |

### Регистр

| Функция | Что делает |
|---------|-----------|
| АА ВЕРХНИЙ | Весь текст в верхний регистр |
| аа нижний | Весь текст в нижний регистр |
| Аа Каждое Слово | Заглавная буква в начале каждого слова |
| Аа предложения | Заглавная буква только в начале каждого предложения |

### Строки

| Функция | Что делает |
|---------|-----------|
| Лишние строки | Сжимает три и более пустых строк до двух |
| Одна строка | Объединяет весь текст в одну строку |
| По предложениям | Каждое предложение на новую строку |
| Убрать нумерацию | Удаляет нумерацию строк вида `1.`, `2.` |

## Система пресетов

Пресеты — пользовательские цепочки операций, применяемые одной кнопкой.

Каждый шаг пресета может быть:
- встроенной функцией форматирования (любой из 15)
- кастомной регуляркой (из библиотеки регулярок)
- набором символов для удаления (из Symbol Bar)

Порядок шагов меняется кнопками ↑↓. Поддерживается экспорт и импорт в JSON.

Хранение: `edit_presets_v1` в localStorage.

---

## Система кастомных регулярок

Вкладка в шите «Пресеты». Каждая регулярка: название, паттерн, флаги (по умолчанию `g`), строка замены. Поддерживает тестирование на текущем тексте прямо в модале. Может быть добавлена как шаг пресета.

Хранение: `edit_regex_v1` в localStorage.

## Вкладка Suno

### Анатомия тега Suno

```
[Verse 1 | rapid-fire flow | dry vocal | trap drums]
 ^^^^^^^^   ^^^^^^^^^^^^^^   ^^^^^^^^^   ^^^^^^^^^^
 структурный    style          style        style
```

Структурные теги: `intro`, `verse`, `chorus`, `outro`, `bridge`, `hook`, `pre-chorus`, `post-chorus`, `breakdown`, `interlude`, `drop`, `build`, `refrain`, `coda`, `fade`, `skit`, `spoken`.

### Only Structure

Убирает все style tags, оставляет только структурные части:

```
До:    [Chorus | melodic rap-sung | strained vocal | heavy drums]
После: [Chorus]
```

Работает по всему тексту. Доступна как шаг в пресетах.

### Edit Tags — chip-редактор

Открывает нижний шит со всеми тегами текста. Каждый тег — карточка:

- Заголовок `[Chorus]` в синем цвете + бейдж `×2` если тег повторяется
- Кнопка `✕` — убрать все style tags у этого тега. `↩` — восстановить
- Строка чипов — каждый style tag отдельным чипом `[melodic rap-sung ×]`

Работа с чипами:
- **Тап на чип** → inline input прямо в чипе, Enter или blur — сохраняет
- **× на чипе** → удаляет только этот style tag
- **+ добавить** → новый style tag к тегу
- Все изменения применяются к тексту мгновенно (live)
- Повторяющиеся теги обновляются все сразу

## Система шаблонов

Заготовки текста с плейсхолдерами `{{имя}}`:

```
Стиль: {{жанр}}, темп {{bpm}} BPM
```

При вставке открывается форма для заполнения. Если плейсхолдеров нет — вставка мгновенная. Поддерживает экспорт и импорт в JSON.

Хранение: `edit_templates_v1` в localStorage.

## localStorage — полная схема данных

| Ключ | Содержимое |
|------|-----------|
| `edit_notes_v1` | `[{name, text, ts}]` — сохранённые заметки |
| `edit_session_hist_v1` | `[{id, text, ts}]` — история сессий, макс. 30 |
| `edit_presets_v1` | `[{name, steps:[...ids]}]` — пресеты |
| `edit_regex_v1` | `[{id, name, pattern, flags, replace}]` — регулярки |
| `edit_templates_v1` | `[{name, text, ts}]` — шаблоны |
